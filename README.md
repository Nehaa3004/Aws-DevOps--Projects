# Projects

## 1. Serverless Full-Stack Web Application Deployment

### Tech Stack
AWS Lambda, API Gateway, DynamoDB, RDS (MySQL), S3, CloudFront, IAM

### Description
- Built and deployed a serverless full-stack web application using AWS services.
- Developed REST APIs using AWS Lambda integrated with API Gateway.
- Used Amazon RDS (MySQL) and DynamoDB for backend data storage.
- Hosted frontend on Amazon S3 with content delivery via CloudFront.
- Configured IAM roles and policies for secure service communication.
- Created deployment and configuration documentation for future reference.

## 2. CI/CD Pipeline for 3-Tier Application Deployment

### Tech Stack
GitHub Actions, Docker, Docker Compose, Kubernetes, AWS EC2, Linux, Nginx, MySQL

### Description
- Designed and implemented an automated CI/CD pipeline for a 3-tier application using GitHub Actions and Docker.
- Automated application build and deployment process on every push to the main branch.
- Built and pushed frontend and backend Docker images.
- Implemented CI/CD workflows using GitHub Actions.
- Managed multi-container deployment using Docker Compose.
- Deployed application on AWS EC2 Linux environment.
- Configured Nginx as reverse proxy for application routing.
- Managed sensitive credentials securely using GitHub Secrets.
- Migrated Docker Compose-based application to Kubernetes multi-node cluster.
- Deployed frontend, backend and database workloads using Kubernetes manifests.
- Worked with Deployments, StatefulSets, Services, ConfigMaps, Secrets and Persistent Volumes.
- Implemented liveness/readiness probes and service discovery mechanisms.
- Performed troubleshooting for container, networking and storage-related issues.
- Reduced manual deployment effort through automation.

## 3. Scalable Web Application Deployment on AWS

### Tech Stack
EC2, Application Load Balancer, Auto Scaling, S3, IAM, CloudWatch

### Description
- Deployed web application on EC2 instances behind Application Load Balancer.
- Configured Auto Scaling Group to maintain availability during traffic changes.
- Implemented security groups and IAM roles for controlled access.
- Configured CloudWatch monitoring and alarms for system performance tracking.
- Performed infrastructure monitoring and troubleshooting activities.
